## 1. SSH checkbox checked by default + smart connection flow ✅

**Goal:** SSH option is enabled by default. If ES port is open, just connect. If closed,
attempt SSH to open it. Provide clear error messages for missing/invalid credentials.

### Changes
- **`index.html`**: SSH checkbox (`connSshEnabled`) now has `checked` attribute by default.
  SSH fields container (`sshFields`) no longer has `d-none` class initially.
- **`app.js`**:
  - `fillForm()`: `ssh_enabled` defaults to `true` if not explicitly set to `false`.
  - `clearForm()`: Passes `ssh_enabled: true` in defaults.
  - `doConnect()`: Enhanced error handling:
    - Detects SSH authentication failures → shows "Check your username and password"
    - Detects missing SSH credentials when ES port is closed → shows "SSH credentials required"
    - Connection refused with SSH enabled but no credentials → prompts user to fill in SSH details

## 2. Preserve filters when changing "Show" count ✅

**Goal:** When user changes the document count (10→500, etc.), any active column filters
should remain applied instead of being cleared.

### Implementation
- **`app.js`**:
  - Added `_preservedIndexFilters = {}` state variable
  - `refreshCurrentIndex()`: Saves current `tableFilters` to `_preservedIndexFilters` before reload
  - `showIndexDetail(indexName, preserveFilters)`: New second parameter; when `true`, restores
    filters from `_preservedIndexFilters` after `captureResults()` would normally reset them
  - After reload: filters are deserialized (handles both Set and Object formats) and re-applied

## 3. Clear All Filters button ✅

**Goal:** Single button to remove all column filters at once.

### UI Location
- Index Detail view toolbar, between pop-out button and Show dropdown

### Implementation
- **`index.html`**: Added button `#btnClearFilters` with `onclick="clearAllFilters()"`
- **`app.js`**:
  - `clearAllFilters()`: Sets `tableFilters = {}`, refreshes tables, updates button state, shows toast
  - `updateFilterButtonState()`: Toggles button styling based on active filters:
    - No filters: `btn-outline-danger` (muted)
    - Has filters: `btn-danger` with badge showing filter count

## 4. Generate Query from Filters button ✅

**Goal:** Convert active table filters into an ES query and navigate to Query Editor.

### UI Location
- Index Detail view toolbar, next to Clear Filters button

### Implementation
- **`index.html`**: Added button `#btnQueryFromFilters` with `onclick="generateQueryFromFilters()"`
- **`app.js`**:
  - `generateQueryFromFilters()`:
    - Collects active filters from `tableFilters`
    - Builds ES query with `bool.must` clauses:
      - Single value → `{ term: { field: value } }`
      - Multiple values → `{ terms: { field: [values] } }`
    - Adds default sort by timestamp desc
    - Populates `#queryBody` textarea with formatted JSON
    - Sets `#queryIndex` to wildcard pattern based on current index (e.g., `adc-network-hourly-*`)
    - Switches to Query view
    - Shows success toast with filter count

## How filter preservation works

```
User clicks Show: 500
    ↓
refreshCurrentIndex()
    ↓ saves tableFilters to _preservedIndexFilters
showIndexDetail(name, true)
    ↓ fetches new data with size=500
captureResults() — resets tableFilters to {}
    ↓
restores tableFilters from _preservedIndexFilters
    ↓
renderResultViews() — applies restored filters
```
